# BalancerPoolToken

*Balancer Labs*

> Highly opinionated token implementation



*- Includes functions to increase and decrease allowance as a workaround   for the well-known issue with `approve`:   https://github.com/ethereum/EIPs/issues/20#issuecomment-263524729 - Allows for &#39;infinite allowance&#39;, where an allowance of 0xff..ff is not   decreased by calls to transferFrom - Lets a token holder use `transferFrom` to send their own tokens,   without first setting allowance - Emits &#39;Approval&#39; events whenever allowance is changed by `transferFrom` - Assigns infinite allowance for all token holders to the Vault*

### decimals

```solidity
function decimals() external view returns (uint8)
```



*Returns the number of decimals used to get its user representation. For example, if `decimals` equals `2`, a balance of `505` tokens should be displayed to a user as `5,05` (`505 / 10 ** 2`). Tokens usually opt for a value of 18, imitating the relationship between Ether and Wei. This is the value {ERC20} uses, unless {_setupDecimals} is called. NOTE: This information is only used for _display_ purposes: it in no way affects any of the arithmetic of the contract, including {IERC20-balanceOf} and {IERC20-transfer}.*
